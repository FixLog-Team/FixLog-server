# AI 기반 트러블슈팅 자동 회고록 (FixLog)

## 프로젝트 개요

본 프로젝트는 **AI 기반 트러블슈팅 자동 회고록 서비스**를 제작하는 것을 목표로 한다.

개발 과정에서 발생하는 각종 오류, 원인 분석, 해결 방법, 참고 자료를

단순 메모 수준이 아닌 **구조화된 문서 자산**으로 축적하는 것을 지향한다.

---

## 왜 이걸 만들까? (문제 인식 및 근거)

- 트러블슈팅 과정은 대부분 메모장, 슬랙, 개인 노트 등 **파편화된 형태**로 기록됨
- 기록을 남기더라도:
  - 다시 블로그나 문서로 정리하는 과정이 번거롭고
  - 시간이 지나면 검색이 어렵고 맥락 복원이 힘듦
- 결과적으로:
  - 같은 문제를 반복해서 조사
  - 과거 경험이 **지식 자산으로 남지 않음**

→ **“기록은 했지만, 활용되지 않는 문제”**를 해결할 필요가 있음

---

## 만들 결과물의 특징 / 차별성 / 우리만의 장점

- 메모장처럼 가볍게 작성해도 **AI가 문서 형태로 자동 정리**
- Notion·블로그 작성 전 단계가 아닌, **기록의 시작점 자체를 서비스화**
- 구글 드라이브와 유사한 **디렉토리 기반 구조**로 장기 관리 가능
- 단순 키워드 검색이 아닌:
  - 트러블슈팅 맥락 기반 검색 지원
- 개인 학습용 + 팀 협업용으로 **확장 가능한 구조**

---

## 핵심 요소 요약

### 핵심 기능

- 트러블슈팅 메모 입력 (자유 형식 / 프롬프트 형식)
- AI 기반 자동 요약 및 문서 구조화
- Markdown 기반 문서 관리
- 전체 문서 대상 검색 기능

### 핵심 서비스 방향

- “단순히 정리된 문서를 만드는 서비스”가 아니라**“정리를 대신 해주고 필요한 기록을 찾아주는 서비스”**

### 예시

- 예전에 해결한 “베트남 소수점 표시” → i18n 베트남 관련 문서 검색
- 퇴사한 개발자가 해당 이슈를 이렇게 해결한 이유

## 일정 예상 (1차 MVP 기준)

- 기간: **2025.12.18 ~ 2026.03.29**
- 1차 MVP 범위:
  - 개인 문서 드라이브
  - Markdown 기반 문서 관리
  - AI 요약 및 정리 기능
  - 문서 단위 및 전체 검색 기능

※ 실제 개발 상황에 따라 일정 및 범위는 조정될 수 있음

---

## 확장 고려 요소

- 개인 드라이브 → 팀 드라이브 확장
- 여러 문서를 대상으로 한 AI 통합 검색
  - git이랑 연결해서, 마크다운 작성하면… 특정 버튼 클릭 시… 잔디 심어지는.. (자동 커밋?)
  - 이건 진짜 그냥 생각일 뿐이므로 나중에 다시 고려 (마크다운 업데이트)
- PDF 다운로드 기능
- Read Only 공유 링크
- 장기적으로는:
  - 기술 위키
  - 팀 회고 아카이브
  - 개인 학습 이력 관리 도구로 확장 가능
    - 하나의 포트폴리오로 활용 가능
